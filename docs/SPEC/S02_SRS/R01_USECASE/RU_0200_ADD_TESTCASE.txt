@title
테스트 케이스 추가 및 SPEC 과의 맵핑

@author
Seokhwan Kim

@date

@desc
어떤 SW 문서화 시스템이든 SPEC 을 명기하게 되어있고, SW 프로젝트에서 SPEC 의
만족여부는 이에 수반하는 테스트의 성공 여부로 판단한다. 예를 들어, 
"어떤 함수 A 는 0 - 100 까지를 입력받는다" 라는 SPEC 이 있는 경우,
테스트 케이스는 -1, 0, 1, 99, 100, 101 을 입력하여, -1 과 101 이 입력되는
경우, invalid 하다는 에러가 나는 것을 보여주어야 한다. (기본적인 경계값 테스트)

이러한 테스트 케이스 역시 모두 왜 이런 테스트가 필요하며 (테스트의 motivation)
또 이러한 필요성을 만족시키기 위해 어떻게 테스트를 기획하였으며 (테스트의 design)
또 어떻게 테스트를 진행할 것인지 (테스트 workflow) 등에 대한 상세한 문서화가 필요하다.

테스트 케이스의 작성이란, 위와 같은 문서를 명세를 하는 것을 의미한다.
